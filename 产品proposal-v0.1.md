## 1. 目标

一个 **手机端应用程序**，AI 作为守秘人（KP）主持克苏鲁的呼唤（CoC 7th）多人跑团。

核心职责：
- 每位玩家用自己手机加入房间
- AI 推进剧情、执行 D100 检定、管理 SAN 变化
- 私密信息（线索、角色卡）仅在各自手机端可见
- 游戏结束后生成复盘总结

## 3. 用户与场景

### 目标用户
想跑团但找不到真人 KP 的新手玩家，或是无人想作为主持人的跑团爱好者。3-5 人小团体，有社交需求。

### 使用场景
几个朋友聚在一起（线下或远程），各自打开 App 进入房间。AI 守秘人主持——描述场景、回应行动、触发检定。每人低头看自己手机上的角色卡和私密线索。公共叙事在聊天区同步。

### 核心路径
```
打开 App → 创建/加入房间 → 选择/创建角色
  → AI 开场叙事 → 玩家自由输入行动
  → AI 回应（叙事 + 可选检定提示）
  → 玩家投骰 → AI 公布结果
  → 循环至结局 → 复盘总结
```

## 4. 规则引擎

CoC 7th 规则由硬编码执行，AI 仅负责叙事。检定判定不走 AI。

### D100 检定

| 投骰结果 | 等级 | 含义 |
|---------|------|------|
| 01 | 大成功 | 完美，额外正面效果 |
| ≤ 技能/5 | 极限成功 | 非常出色 |
| ≤ 技能/2 | 困难成功 | 高质量完成 |
| ≤ 技能值 | 常规成功 | 完成 |
| > 技能值 | 失败 | 未达成 |
| 96-100（技能<50） | 大失败 | 灾难性后果 |

### SAN 机制
- 起始 SAN = POW
- 遭遇神话事件损失 SAN（1/1D3 至 1D10/1D100）
- 单次损失 ≥5 → 暂时性疯狂
- SAN = 0 → 永久疯狂，角色退出

### 衍生属性
- HP = (CON + SIZ) / 10
- MP = POW / 5
- LUCK = 3D6 × 5

## 6. AI 交互（方案阶段，待联调验证）

每次请求发给 AI 的上下文：
- 玩家输入
- 当前场景（地点、时间、在场 NPC）
- 角色状态（HP/SAN/持有物品）
- 最近 6 轮对话历史
- 已获取的线索摘要

AI 预期返回 JSON：
```json
{
  "narration": "铁门在生锈的铰链上发出一声刺耳的呻吟…",
  "skill_check": {
    "triggered": true,
    "skill": "spot_hidden",
    "skill_name": "侦察",
    "target": 65,
    "reason": "你注意到脚印的排列有些奇怪"
  },
  "san_change": null,
  "private_info": [
    {"player": "p1", "content": "你在门框缝隙发现了一片灰色布料"}
  ],
  "suggestions": ["检查脚印", "进入会客室", "上楼"]
}
```

叙事由 AI 生成，`skill_check` 由后端规则引擎硬编码判定。两者解耦。

## 7. 技术架构

### 技术选型

| 层 | 选择 | 简要理由 |
|---|------|---------|
| 前端 | React Native (TS) | 跨 iOS/Android，团队已掌握 TS |
| 后端 | Python FastAPI | AI SDK 生态（DeepSeek/Claude 均为 Python 优先），WebSocket 原生支持 |
| 数据库 | PostgreSQL | 模组数据结构固定，关系型查询更合适 |
| AI 模型 | DeepSeek / Claude（待定） | 需实测：成本、延迟、CoC 叙事质量三者权衡 |

### 架构分层
```
React Native 前端 (TS)  ← 手机端 UI
        │
FastAPI 后端 (Python)   ← HTTP + WebSocket
   ┌────┴────┐
规则引擎    AI 服务     ← 硬编码检定 / LLM 调用
   └────┬────┘
   PostgreSQL           ← 角色/模组/会话
```

### 多人房间
WebSocket 维持房间内玩家实时同步：
- 公共叙事广播到全员
- 检定请求发给特定玩家
- 私密信息仅推送目标玩家

## 8. MS1 验收标准（草案）
- 打开 App → 创建/加入房间 → 开场叙事
- 至少 3 轮"输入 → AI 回应 → D100 检定"完整循环
- AI 不犯规则错误（如要求投 D20、混淆检定等级）
- 前端核心屏幕串联，无阻断性 bug
