## 1. 讨论回顾

两场会议围绕产品形态、技术架构、交互方式三个层面展开了多轮讨论。

### 1.1 产品形态

讨论中浮现了三种思路：

- **平台化**：做一个可复用多规则（D&D、CoC、狼人杀等）的平台，支持玩家与创作者身份互通，形成内容生态
- **AI 主持人工具**：聚焦"帮新人跑团"，解决"想玩但没人带"的痛点。高玩对 AI DM 需求不大，新手才是核心用户
- **社交角色扮演**：抽象出"角色扮演"和"社交"两个核心，无论新手高玩都能获得纯粹的角色扮演体验

### 1.2 产品形态：已共识

经过讨论，团队在以下方向上达成一致：

- **手机端应用程序**：React Native（TypeScript）开发，每位玩家使用自己的手机进入房间。后端 Python FastAPI
- **多人房间模式**：玩家输入房间码加入房间，AI 作为守秘人主持游戏
- **聚焦 CoC 规则首发**：首版只做克苏鲁的呼唤，其他规则（D&D、狼人杀、血染钟楼、剧本杀）作为后续扩展

### 1.3 交互方式

- **线下多人模式**：每人在自己手机上打开 App 加入同一房间。AI 的公共叙事在聊天区同步展示，角色卡、私密线索、个人检定在各自手机端独立显示——同处一室时每人低头看自己的手机，屏幕内容互不可见
- **语音输入**：讨论了声纹识别实现开放麦语音输入的可能性，但首期优先文字输入，确保核心流程先跑通
- **结构化输入辅助**：输入栏可提供常用动作类型选择（行为、说话、侦查等），降低新手门槛，同时帮助 AI 区分玩家意图

### 1.4 技术架构

- **技术选型**：前端 React Native（TypeScript）+ 后端 Python FastAPI
- **三层架构**：通用框架（core，掷骰、交互等） / CoC 规则插件（检定、SAN 机制等） / 模组层（场景、NPC、物品）
- **多 Agent 还是单 Agent**：讨论了多 Agent 方案（剧情编排、玩家行为处理、旁白、合规审查各由一个 Agent 负责），但也提出了多 Agent 之间"幻觉"和泄密风险的担忧。下午会议倾向"先单 Agent 验证，再拆分"
- **RAG + 硬编码结合**：规则检索走向量检索，关键计算（检定判定、数值结算）硬编码保证准确
- **模组数据结构**：场景（静态/时序）、NPC、物品三大要素。关键物品由模组创作者手动标注重要性

## 2. 用户与场景

**目标用户**：想跑团但找不到真人 KP 的新手玩家。他们有社交需求——几个人想一起玩——但缺少一个有经验的主持人带团。

**核心使用场景**：
- 2-4 个朋友聚在一起（线下或线上），各自打开手机 App 进入同一个房间
- AI 守秘人主持游戏，推进剧情，执行规则判定
- 每人手机端查看自己的角色卡、私密线索、进行个人检定
- 公共叙事在聊天区同步展示

---

## 3. 核心问题

跑团的瓶颈在于 **KP 稀缺**。CoC 守秘人需要同时做到：
1. 正确执行 D100 检定规则和 SAN 机制
2. 即兴叙事回应玩家的任意行动
3. 管理多人场景下的私密信息分发

AI 恰好能同时处理"规则执行"和"自由文本叙事"，解决了新手群体"想玩但没人主持"的核心痛点。

## 4. 产品形态

一个**手机端应用程序**，AI 作为 CoC 守秘人主持多人跑团。

**技术栈**：React Native（TypeScript）+ Python FastAPI 后端

**核心交互流程**：
打开 App → 创建/加入房间 → 分配角色 → AI 开场叙事 → 玩家自由输入行动 → AI 回应（含检定提示）→ 投骰 → 结果公布 → 叙事推进 → 到达结局

**交互原型参考**：团队已基于 HTML 完成概念原型探索，包含 7 屏完整流程：登录、游戏选择、世界选择、剧情导入、角色创建、等待房间、游戏对话。交互设计以手机竖屏为核心画板。正式开发将用 React Native 重建，概念原型的交互逻辑作为设计参考。

**已共识**：
- React Native 手机应用（非网页）
- 多人房间制（创建房间 → 分享房间码 → 加入 → AI 主持）
- CoC 7th 规则首发
- 文字叙事为主，结构化信息（状态栏、技能列表、地图）辅助展示

**待定**：
- 语音输入的首期范围
- 多 Agent 还是单 Agent

## 5. 不做的事

| 不做 | 理由 |
|------|------|
| 桌面端 / 网页版 | 议题原始设定即为手机端参与，首期聚焦移动端 |
| D&D / 狼人杀 / 血染钟楼 / 剧本杀（首期） | 一个规则未验证就铺多个，必定烂尾 |
| AI 替补玩家 / AI NPC 队友（首期） | 议题进阶目标第 3 条，非 MVP 范围 |
| AIGC 图片生成 | 成本高，对核心跑团体验非决定性 |
| UGC 模组创作工具 | 无用户基础前无人使用 |
| 付费订阅与商城 | 首发免费 |
| PDF/Word 模组直接导入 | 解析难度高，首期手工录入 |
| 完全无边界沙盒 | 预置模组提供叙事边界 |
