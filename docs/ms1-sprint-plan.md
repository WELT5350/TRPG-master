# MS1 冲刺：一周路演计划

> 目标：前端连真实后端，跑通一条完整链路 → 路演可用

---

## 核心链路

```
创建房间 → 选择模组 → 进入游戏 → 输入行动 →
AI 生成叙事 → 玩家掷骰 → 规则引擎判定 → 结果显示
```

---

## 任务分配

| 优先级 | 任务 | 负责人 | 预估 | 产出 |
|--------|------|--------|------|------|
| **P0** | FastAPI 后端骨架 | `@待分配` | 2-3 天 | `/auth/create-room` + `/room/create` + `/game/:roomCode/action` 3 个接口 |
| **P1** | AI 叙事串联 | `@待分配` | 2-3 天 | LLM 调用 + Prompt 模板 + JSON 解析，接入 `/game/action` |
| **P2** | 规则引擎 | `@待分配` | 半天 | D100 检定函数，接入 `/game/check` |
| **—** | 前后端联调 | 全员 | 半天 | 前端切 `VITE_MOCK_API=false`，串通整条链路 |

---


P0 和 P1 完全并行，互不依赖。P2 插空做，联调在 D6 进行。

---

## 做完的判定

- [ ] 前端 `VITE_MOCK_API=false` 后，能走通登录→创建房间→输入行动→AI 叙事
- [ ] 掷骰后能看到 D100 判定结果
- [ ] AI 返回合法 JSON，不含判定结果（AI 不可宣布骰子结果）
- [ ] 全场无报错崩溃
