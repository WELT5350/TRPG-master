## 动机 / 用户故事

AI 叙事是产品核心体验。如果 AI 不能产出合格的克苏鲁恐怖/悬疑叙事，整个产品就不成立。但 AI 从哪里来——云端大模型还是本地部署——决定了成本、延迟和质量。

## 目标用户

产品决策者、后端架构设计者。这个选择影响预算、响应时间和叙事质量。

## 现有做法及其不足

| 方案 | 描述 | 优劣 |
|------|------|------|
| **A. 云端大模型（选中）** | 调用 API（具体模型待实测选定） | ✅ 叙事质量高、迭代快 ❌ 有 API 成本、依赖网络 |
| B. 本地开源模型 | 自部署，离线可用 | ✅ 零 API 成本 ❌ 中文 CoC 恐怖叙事质量未经验证，部署运维成本高 |
| C. 固定剧本（无 AI） | 预设分支文本 | ✅ 零成本、确定性高 ❌ 无法应对自由输入，失去核心价值 |

**选择 A 的依据：** AI 叙事是核心差异化。如果本地模型的中文克苏鲁叙事质量不达标，产品从根上就不成立。云端大模型是目前唯一被验证可以达到要求的路径。

> 竞品参照：AI Dungeon 使用 OpenAI 系模型通过 prompt 工程约束叙事风格；我们的需求介于开放叙事和 NPC 角色一致性之间。

## 重要说明

具体选哪个模型（DeepSeek / Claude 等）属于技术实现决策，**不在本提案范围**。将在架构设计阶段通过实测（叙事质量 + 成本 + 延迟）选定。

MS1 不进行模型微调。

## 验收标准

- [ ] AI 产出的中文叙事符合 CoC 恐怖/悬疑风格，不偏离跑团语境
- [ ] AI 不犯基础规则错误（如要求投 D20、混淆检定等级、越权宣布结果）
- [ ] 单次 AI 响应延迟 < 8 秒（含网络）
- [ ] 模型选定方案通过团队评审
