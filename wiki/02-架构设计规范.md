# 架构设计规范

## "代码即架构"

MS1 必须产出**空壳代码**，含真实接口和 Mock 数据，提交到仓库，编译通过，演示模块连通性。

### 前端

- 封装 API 调用到 SDK 风格的 model 层
- 定义 TypeScript 类型
- 用 Mock 数据驱动交互流程

### 后端

- 定义包结构、接口、数据结构
- 方法用 stub 实现
- 接口定义即契约

## 设计步骤

1. 需求分析（以产品决策为输入）
2. 风险分析
3. 模块分解 + 接口定义
4. 关键模块详细设计

## 产出物

1. **骨架代码（主要）** — 空壳但可编译运行
2. 架构描述文档（可选）
3. 复杂模块的详细设计文档

## 架构演进追踪

每个里程碑记录架构变化及原因。警告信号：
- 频繁大型重构
- 架构无法支撑特性交付

## 人类主导设计

系统架构必须由人类设计，不可由 AI 生成粗糙实现。
